PROJECT CONTEXT: EDGE-BASED POLYFORM ENGINE (MONOMINO, WITH HOLES)

Polyominoes are generated using edge geometry with ordered
boundary cycles. Cell-based representations are not used.

------------------------------------------------------------

CURRENT STATUS

- Monomino generator complete (hole mode)
- Matches OEIS A000105
- Verified through n >= 12
- Hole counts:
  n = 7 -> 1
  n = 8 -> 6

------------------------------------------------------------

CORE REPRESENTATION

Poly:
- one outer cycle
- zero or more hole cycles

Cycle:
- ordered list of lattice vertices
- unit-length edges
- no duplicate vertices within a valid cycle

------------------------------------------------------------

KEY INVARIANTS

- All edges are unit length
- Cycles are ordered
- Poly is a set of simple cycles
- No floating point arithmetic

- Outer = maximum absolute signed area
- Outer orientation = CCW
- Hole orientation = CW
- Canonical form = [outer] + sorted(holes)

- All cycles transform together

------------------------------------------------------------

ATTACHMENT MODEL

- Align antiparallel edges
- Cancel antiparallel overlaps
- Reject parallel overlaps
- Walk remaining edges into cycles

Ambiguous vertex rule:
- If duplicate vertex appears during traversal,
  retry with opposite turn choice

- Keep all resulting cycles

------------------------------------------------------------

CRITICAL TOPOLOGY RULE

Duplicate vertices inside a cycle indicate incorrect
turn resolution.

Correct behavior:
- retry alternate turn
- accept configuration if resulting cycles are valid

------------------------------------------------------------

CANONICALIZATION

Performed once:

attach → extract cycles → build Poly → canonicalize → store

Procedure:
- apply symmetry to all cycles together
- normalize translation once
- identify outer by maximum area
- enforce orientations (outer CCW, holes CW)
- canonicalize cyclic shifts
- keep outer first
- sort holes only
- choose lexicographically minimal Poly

Do NOT sort all cycles together.

------------------------------------------------------------

FRONTIER RULE

Frontier includes edges from all cycles:
- outer
- holes

------------------------------------------------------------

DUMP FORMAT

mono_dump:

<0|1> cycle0 | cycle1 | ...

- outer cycle prints first
- 1 indicates presence of holes
